## 1. UX Mobile — 6.5/10

### Critical
- **Форма в самом низу страницы** — пользователь с рекламы (Instagram/VK) проходит 6+ экранов до заявки
- **Нет sticky/floating CTA** после hero — на mobile теряется главное действие при скролле
- **Меню без блокировки скролла body** — при открытом бургере фон прокручивается
- **Якорные ссылки перекрываются fixed header** — нет `scroll-padding-top` на `html`

### Recommendations
- Добавить фиксированную кнопку «Записаться» внизу экрана (mobile only, `position: fixed; bottom: 0`)
- Укоротить hero H1 на mobile или использовать `clamp` с max 3 строками
- Разделить поле «Телефон или email» на два таба или `input type="tel"` с переключателем
- Добавить `tel:` ссылку «Позвонить» как альтернативу форме
- Уменьшить вертикальные отступы секций на `<768px` (`padding: 3rem 0`)
- Pre-select курс в форме при клике CTA из карточки (query/hash `#contact?course=consulting`)

### Quick wins
- `html { scroll-padding-top: var(--header-h); }`
- `body.nav-open { overflow: hidden; }` при открытом меню
- `inputmode="tel"` и `autocomplete="tel"` для телефона

## 2. UX Web — 7/10

### Critical
- **Форма имитирует отправку** — success-сообщение без реальной доставки заявки разрушает доверие при повторном визите
- **Telegram «скоро»** виден пользователю — лучше скрыть до готовности
- **Несогласованность бренда:** title «PsyConstructor», repo `design_yourself`, logo «PsyConstructor»

### Recommendations
- Добавить **сравнительную таблицу курсов** (длительность / документ / для кого / цена) — ускорит выбор
- Вынести **один primary CTA** в header + дублировать после блока курсов (mid-page conversion)
- Добавить **микрокопию под формой:** «Ответим в течение 24 часов»
- Отзывы: добавить фото (или инициалы в круге), город, программу которую прошли
- Footer: ссылки «Политика конфиденциальности», «Оферта», email для связи
- При клике «Смотреть программы» — smooth scroll ok, но добавить highlight на секцию courses

### Quick wins
- Скрыть кнопку Telegram до заполнения `href`
- Единый slug бренда в title, logo и og:title
- Hover-состояние на карточках курсов (`translateY` + shadow)

## 3. SEO — 5/10

### Critical
- **`og:image` относительный** (`./assets/images/hero.png`) — при шаринге в VK/Telegram картинка **не отобразится**
- **Нет `og:url` и canonical** — дубли и некорректная атрибуция трафика
- **Нет structured data** — упущены rich results для FAQ и курсов

### Recommendations
- Заменить meta на абсолютные URL:
  ```html
  <link rel="canonical" href="https://stengachanga.github.io/design_yourself/">
  <meta property="og:url" content="https://stengachanga.github.io/design_yourself/">
  <meta property="og:image" content="https://stengachanga.github.io/design_yourself/assets/images/hero.png">
  <meta name="twitter:card" content="summary_large_image">
  ```
- Добавить JSON-LD:
  - `Organization` (название, url, logo)
  - `Course` × 4 (name, description, offers/price)
  - `FAQPage` (вопросы из секции FAQ)
- Title: включить geo/формат — «Курсы гештальт-терапии онлайн — обучение с супервизией»
- Оптимизировать hero.png (<200 KB WebP), добавить `width/height` везде
- `font-display: swap` уже через Google Fonts — рассмотреть self-host для LCP
- Создать `robots.txt` и `sitemap.xml` (одна страница — всё равно полезно)

### Quick wins
- Абсолютные OG-теги (5 минут, высокий impact)
- FAQ schema — часто даёт расширенный сниппет в Google

## 4. Design — 7/10

### Critical
- **Одна иллюстрация `instructor.png` используется дважды** — визуальная монотонность, ощущение «заглушки»
- **Животные-«психологи»** — тёпло и запоминается, но часть аудитории воспринимает как несерьёзно для переподготовки

### Recommendations
- Создать **2–3 варианта иллюстраций** или добавить блок с реальными фото преподавателей
- Разработать **логотип** (не только текст PsyConstructor)
- Иконки для блока «Преимущества» (practice, supervision, flexibility, community)
- На desktop 4 колонки карточек — **разная высота** ломает ритм; использовать `grid-auto-rows` или flex equal height
- Badge «Популярный» на mobile может наезжать на заголовок — добавить `padding-right` на h3
- Проверить контраст CTA `#5a7d6a` на белом — возможно усилить до `#456352` для текста на светлом фоне
- Добавить subtle **section dividers** или номера шагов (1→2→3→Запись)

### Quick wins
- Уникальное изображение для блока «Преподаватели» (не hero/instructor duplicate)
- `:focus-visible` стили для всех интерактивных элементов (a11y + polish)
- Favicon в `.ico` + apple-touch-icon

## 5. Psychologist — 6/10

### Critical
- **Нет чекбокса согласия на обработку ПДн** — форма собирает имя и контакт (152-ФЗ)
- **«Международная аккредитация» без указания организации** — неконкретное заявление снижает доверие у профессионалов
- **Отзывы без контекста программы** — «Мария, практикующий психолог» — какой курс?

### Recommendations
- Добавить блок **«Для кого этот курс»** vs «Не подойдёт, если…» — снижает неподходящие заявки
- Уточнить в FAQ:
  - Нужна ли личная терапия в процессе обучения?
  - Сколько часов супервизии включено?
  - Чем «гештальт-консультант» отличается от «психолога»/«психотерапевта» в РФ?
  - Есть ли клинические противопоказания для self-work на курсе?
- Указать **аккредитацию по имени** (EAGT, ВООПИГ, конкретный институт) или убрать claim
- Добавить **этический дисклеймер:** «Обучение не заменяет медицинскую/psychiatric помощь»
- Для переподготовки: уточнить **лицензируемость**, соответствие профстандарту, часы практики
- Tone: заменить «наконец поняла» в отзыве на более нейтральное (меньше обещаний результата)

### Quick wins
- Чекбокс «Согласен на обработку персональных данных» + ссылка на политику
- FAQ: «Нужна ли личная терапия?» — стандартный вопрос для gestalt training
- Подпись под ценами: «Не является публичной офертой»
