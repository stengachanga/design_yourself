## Context

Проект `PsyConstructor` — новый репозиторий без существующего кода. Требуется маркетинговый лендинг для рекламы психологических курсов с уклоном в гештальт-терапию, с аналитикой и публикацией на GitHub Pages. Целевая аудитория — русскоязычные пользователи, приходящие из рекламных каналов (соцсети, контекст).

Ограничения:
- Статический хостинг (GitHub Pages) — без серверного бэкенда
- Минимальная сложность сборки и поддержки
- ID Яндекс.Метрики не должен попадать в публичный репозиторий в открытом виде (использовать GitHub Secrets + подстановка при сборке)

## Goals / Non-Goals

**Goals:**
- Быстрый, адаптивный одностраничник с понятной структурой и CTA
- Интеграция Яндекс.Метрики с целями для оценки рекламы
- Автоматический деплой на GitHub Pages при push в `main`
- Лёгкая замена контента (тексты, изображения) без переписывания логики

**Non-Goals:**
- CMS, админ-панель, многостраничный сайт
- Серверная обработка форм (на первом этапе — mailto, внешняя форма, или Formspree/Getform)
- Личный кабинет, оплата, CRM-интеграция
- Google Analytics (только Яндекс.Метрика по требованию)

## Decisions

### 1. Стек: чистый HTML + CSS + vanilla JS

**Решение:** Статический сайт в папке `src/` (или корень), без React/Vue.

**Обоснование:** Лендинг простой, GitHub Pages отдаёт статику напрямую. Нет необходимости в тяжёлом фреймворке.

**Альтернативы:** Vite + статический экспорт — избыточно для одной страницы; Jekyll — лишняя зависимость от Ruby.

### 2. Структура файлов

```
/
├── index.html          # или src/index.html → копируется в docs/
├── css/
│   └── styles.css
├── js/
│   ├── main.js         # навигация, форма, scroll goals
│   └── metrika.js      # инициализация счётчика
├── assets/
│   └── images/
├── docs/               # публикуемая папка для GitHub Pages (из main)
└── .github/workflows/
    └── deploy.yml
```

**Решение:** Публиковать из `/docs` на ветке `main` — проще, чем отдельная ветка `gh-pages`.

**Альтернатива:** GitHub Actions с `actions/upload-pages-artifact` — предпочтительнее для подстановки секретов (Metrika ID) при сборке.

### 3. Деплой: GitHub Actions + GitHub Pages (modern)

**Решение:** Workflow `deploy.yml`:
1. Checkout
2. Подстановка `YANDEX_METRIKA_ID` из secrets в шаблон `metrika.js` или `index.html`
3. Копирование/сборка в `docs/` или artifact
4. Deploy via `peaceiris/actions-gh-pages` или встроенный `actions/deploy-pages`

**Обоснование:** Секреты Metrika не в git; автодеплой при push.

### 4. Аналитика: Яндекс.Метрика

**Решение:**
- Счётчик через официальный snippet в `<head>`
- Цели через `ym(counterId, 'reachGoal', 'goal_name')` в обработчиках кликов и Intersection Observer для scroll goals
- Webvisor + clickmap включены в init-параметрах

**Конфиг:** `YANDEX_METRIKA_ID` в GitHub Repository Secrets; локально — `.env.local` (в `.gitignore`).

### 5. Форма заявки

**Решение (MVP):** Клиентская форма с валидацией; отправка через бесплатный form backend (Formspree) или ссылка на Telegram/WhatsApp. После успешной отправки — `reachGoal('form_submit')`.

**Альтернатива:** Только кнопка «Написать в Telegram» — проще, но хуже для конверсионной аналитики формы.

### 6. Дизайн и UX

**Решение:**
- Спокойная цветовая палитра (теплые нейтральные + акцент для CTA) — соответствует психологической тематике
- Системные шрифты или Google Fonts (e.g., Inter + serif для заголовков)
- Mobile-first CSS с `clamp()` для типографики
- Плавный scroll (`scroll-behavior: smooth`), фиксированный header с якорями

### 7. SEO

**Решение:** Семантический HTML5 (`header`, `main`, `section`, `footer`), meta description, Open Graph, `lang="ru"`, favicon.

## Risks / Trade-offs

| Риск | Митигация |
|------|-----------|
| Form backend без бэкенда — спам, лимиты | Formspree с honeypot; или Telegram CTA как fallback |
| Metrika ID в исходниках | Подстановка через CI secrets |
| GitHub Pages project URL — сломанные пути | Относительные пути `./css/`, `./js/`; не абсолютные от корня домена |
| Placeholder-контент снижает доверие | Чётко пометить в README, что тексты заменяются перед запуском рекламы |
| Webvisor — вопросы приватности | Упомянуть в политике конфиденциальности (будущая задача) |

## Migration Plan

1. Создать структуру проекта и базовый HTML/CSS
2. Добавить контент-заглушки на русском
3. Подключить Metrika (локально с тестовым ID)
4. Настроить GitHub repo, Secrets, Actions workflow
5. Включить GitHub Pages в настройках репозитория
6. Проверить деплой, цели в Metrika, мобильную вёрстку
7. Заменить заглушки реальным контентом перед рекламной кампанией

**Rollback:** Откат через revert коммита в `main`; GitHub Pages обновится автоматически.
